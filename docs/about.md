# About Aetheria

Hey! Thanks for using Aetheria's Skyblock Mod. This MD covers two things: what data the mod may collect and how the license works. Nothing scary, we just want to be upfront about it.

---

## Privacy

### First Launch

When you launch Aetheria for the first time, you'll see a **Privacy Notice screen** before the mod makes any external connections. You get to review everything and opt out of anything you're not comfortable with before it happens.

> No data leaves your client until you've acknowledged this screen.
>
> Skyblock recipe data and the cape index may be fetched from GitHub before the privacy screen appears, since these are needed for core features to load.

### What we collect

Analytics are **fully optional**. Here's what we may collect if you have them on:

| Data | Why |
|------|-----|
| Your Minecraft username | To count unique players |
| Your mod list | To identify your Aetheria version and help with crash reports |

We don't collect passwords, chat messages, files, server IPs, or anything else.

You can turn analytics on or off at any time under **Settings > Privacy**.

### Feature data

Some features need to send a small amount of data to work properly. For example, the capes feature sends your username and selected cape to our backend so other Aetheria users can see your cape in-game. We only ever send the minimum needed for a feature to function.

All of these features are **opt-in**, and each one can be toggled individually in settings.

### Who sees your data

Only the project's Original Author has access to any collected data. It is never sold or shared with third parties.

### Third-party infrastructure

Some of our backend services run on infrastructure provided by [Vercel](https://vercel.com/legal/privacy-policy) and [Cloudflare](https://www.cloudflare.com/privacypolicy/). These providers may log things like your IP address as part of their normal network operations, which is outside our control. Feel free to check out their privacy policies if you want to know more.

## License

Aetheria is licensed under the **Aetheria Mod License v1.0 (AML-1.0)**. The source code is publicly available and we're committed to keeping it that way, not just as a project promise but as an actual requirement of the license itself.

Anyone is free to:

- View and study the code
- Fork the repository
- Modify the source code
- Submit contributions

However, AML-1.0 has a few differences from common open-source licenses:

- Compiled builds (like `.jar` files) cannot be publicly redistributed without permission from the Original Author
- Official releases should be obtained from the project's official distribution channels
- Derivative works based on Aetheria's code must follow AML-1.0 and keep the relevant source code publicly available
- The Aetheria name, logos, and branding cannot be used for unofficial projects without permission

### Our commitment to open availability

Aetheria is committed to always keeping its source code publicly accessible. This is not only a project promise but also a hard requirement under Section 5 of AML-1.0, which requires all covered source code to remain publicly available at all times. This requirement applies to all parties, including the Original Author. No covered source code will ever be intentionally made private or hidden.

The only exceptions are sensitive items like credentials, API keys, environment files, and private configuration data, which are excluded from this requirement.

Here's a quick summary of what this means for you:

### For players

- Download and use Aetheria normally
- Use it in modpacks, videos, streams, and on servers
- No permission needed for personal gameplay

### For content creators

- Make videos, streams, guides, and reviews freely

### For contributors

- By contributing, your work becomes part of the project under AML-1.0
- You keep ownership of your contributions while allowing the project to maintain and distribute the combined work

### For developers

- You can inspect, fork, and modify the source code freely
- Publicly distributing compiled builds requires permission from the Original Author
